# Installable Mirror Package Contract

Active Mirror should be installable as one personal mirror that can run through a
browser, local machine, cloud Worker, or hybrid setup without changing identity.

## Package Pieces

- `mirror kernel` - applies Mirror Loop v1, identity rails, truth gates, output
  straitjacket, and receipts.
- `transparent router` - chooses local, cloud, or tool workers and records why.
- `council control plane` - routes broad intent through thread, source, runtime,
  ops, design, security, state, or promotion ownership before promotion.
- `MirrorDash Glass` - exposes provider/model/tool route, memory scope, prompt
  disclosure, source policy, fail-safe state, and remaining opacity.
- `vault adapter` - compiles approved context packets from local memory; raw
  vault access is forbidden.
- `source adapter` - invokes whitelisted web/source tools and returns citations
  or `needs_checking`.
- `memory gate` - proposes memory writes as candidates; user or policy approval
  promotes them.
- `training firewall` - allows only approved mirror examples with receipts,
  consent, evals, and rollback.
- `fail-safe` - disables model/tool egress and returns deterministic guarded
  output.

## Modes

| Mode | Runs where | Allowed use |
| --- | --- | --- |
| Browser-only | User browser plus gateway | First-run mirror, local preferences, public gateway receipts. |
| Local-first | Mac/mini/phone runtime | Private vault context, local model candidates, signed local memory. |
| Cloud-assisted | Worker plus hosted providers | Public/source/tool routing with no browser secrets. |
| Hybrid | Local control plane plus cloud edge | Local identity and memory with cloud source/model workers when approved. |

## Non-Negotiables

- Active Mirror is the identity; models are workers.
- The council route chooses the accountable owner before promotion.
- The mirror is the filter.
- Current facts require source-check or `needs_checking`.
- Web access is tool-routed and receipt-backed, not ambient model browsing.
- Raw vault data never routes directly to a model, source tool, or trainer.
- Memory is signed, scoped, user-owned, and revocable.
- LoRAs/adapters remain candidate workers behind the same gates.
- Fail-safe must be able to cut off egress without breaking the user surface.
