# Market Position v0

Last refreshed: 2026-06-23.

## Hard Conclusion

The market is not moving toward one smarter chatbot. It is moving toward agent operating layers.

That means Active Mirror should not position itself as "an orchestrator." Multi-agent orchestration is becoming infrastructure: model/API providers, cloud platforms, workspace suites, and local agent tools are all moving there.

## Active Mirror Position

```text
Active Mirror is the trust layer for AI agents.
It gives models memory, permissions, evaluation, rollback, and receipts before they act.
```

## Why This Is Defensible

| Market movement | What it means | Active Mirror response |
| --- | --- | --- |
| Multi-agent orchestration as API | Orchestration alone becomes commodity infrastructure | Sit above orchestration as trust/control plane |
| Enterprise agent platforms | Google, Microsoft, and OpenAI are adding governance and agent operations | Make governance user-visible and local-first |
| Workplace agent benchmarks | Agents still fail on real tools, artifacts, cost, runtime, and long-horizon workflows | MirrorEval and MirrorBench become core primitives |
| Local-first memory systems | Memory is moving from convenience to governance | Default Ledger and controlled memory writes |
| MCP/tool security | Tool names, prompts, and memory can become attack surfaces | ToolGraph, approval gates, source discipline, and receipts |
| Design/artifact engines | Figma, Open Design, and Comfy can generate surfaces and assets | Govern artifact engines instead of replacing them |

## Product Primitive

Not:

```text
Memory.
```

Better:

```text
Reusable judgment plus verified action.
```

## Product Stack

```text
Human intent
  -> Reflective task framing
  -> Consent boundary
  -> Default Ledger
  -> ToolGraph route
  -> AMOS execution
  -> MirrorEval
  -> Receipt
  -> Optional memory promotion
  -> Reset or rollback if needed
```

## What To Ignore

- Do not chase sovereign frontier-model building.
- Do not sell reflection as a vague concept.
- Do not build a generic chatbot.
- Do not merge Active Mirror / AMOS with client-specific systems.
- Do not expose a flat MCP/tool list without ToolGraph governance.
- Do not let memory write automatically without provenance and expiry.
- Do not let agent work proceed without fail-before/pass-after acceptance tests.
