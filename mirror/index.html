Browser runtime

Your mirror opens as a private workspace, not a blank chatbot.

The runtime keeps context local first, reflects before routing, and turns each approved turn into a visible viewport with receipts.

How a turn works

  • 1. Capture the prompt and local context.
  • 2. Reflect and prune sensitive details before routing.
  • 3. Generate a viewport, artifact, or next action.
  • 4. Save only approved memory with a receipt.

Model route

The current public runtime is model-agnostic. Local deterministic reflection runs first. A small local helper can classify, redact, and triage. Gemini or other frontier models are optional workers behind consent gates.
