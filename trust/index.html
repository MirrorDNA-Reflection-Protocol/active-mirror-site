Security posture for the browser runtime

The target architecture stays conservative: secrets never reach the frontend, providers sit behind a gateway, and local state has explicit gates.

No frontend keysAPI keys belong behind Workers or server-side gateways, never in browser JavaScript.
No silent memoryMemory promotion is a user-visible decision with receipt context.
Provider isolationExternal models receive scoped tasks, not the whole vault by default.
Failure remains usefulIf a model route fails, the browser can still generate a deterministic local board.
