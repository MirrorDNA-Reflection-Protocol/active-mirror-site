Privacy starts with less collection.

Last updated June 21, 2026. This page describes the public-site and early-product privacy posture.

Public site

The public site is designed as a static front door. It does not need personal details to explain the product or run the browser demo.

Browser mirror

The current browser demo can generate a local board without sending your input to a model provider. Hosted model routes should be disclosed before use.

Model routes

When hosted routes are enabled, provider calls should happen through a server-side gateway. API keys must not be exposed in frontend code.

Memory

Private context should not become durable memory unless the user approves the memory decision. Receipts should show what was saved, ignored, or excluded.

Contact

Questions about privacy can be sent to hello@activemirror.ai.
